## Design Context

### Users

Furnace is for terminal-native software engineers using an agent for long, tool-heavy coding sessions. The interface must keep the prompt, model activity, permissions, queued work, and session telemetry legible without slowing keyboard-first operation.

### Brand Personality

Capable, experimental, and unapologetically technical. Furnace should feel like a purpose-built instrument rather than a generic chat client, while staying calm enough for hours of daily use.

### Aesthetic Direction

The product supports multiple intentionally different interface compositions instead of pretending color themes are layouts. Directions can range from editorial minimalism to dense command-center instrumentation, but each must preserve terminal readability, narrow-width fallbacks, and the user's existing theme.

### Design Principles

1. Change hierarchy and workflow—not only color—when introducing a layout.
2. Preserve keyboard flow and live session state through every visual transition.
3. Use terminal characters as structure and information, not ornamental noise.
4. Keep dense telemetry optional; the coding conversation remains primary.
5. Make every composition responsive to narrow terminals and respectful of reduced visual complexity.
